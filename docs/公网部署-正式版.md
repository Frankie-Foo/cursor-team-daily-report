# 公网 POST — 正式版（Cloudflare Zero Trust 固定域名）

同事 **不需要 VPN**。API 通过 **HTTPS 公网地址** POST 入库。

---

## 一、主管一次性配置（约 5 分钟）

### 1. Cloudflare Zero Trust 创建隧道

1. 打开 [Cloudflare Zero Trust](https://one.dash.cloudflare.com/) → **Networks** → **Tunnels**
2. **Create a tunnel** → 选 **Cloudflared** → 名称：`cursor-team-reports`
3. **Public Hostname** 添加一条：
   - Subdomain：`cursor-reports`（或任意）
   - Domain：选你在 Cloudflare 上的域名（如 `example.com`）
   - Service：`http://127.0.0.1:8080`
4. 复制页面上的 **Tunnel Token**（一长串）

### 2. 写入本机 `.env`

在 `D:\cursor-team-daily-report\.env` 增加/修改：

```env
REPORT_API_URL=https://cursor-reports.你的域名.com
API_HOST=127.0.0.1
API_PORT=8080
CLOUDFLARE_TUNNEL_TOKEN=粘贴TunnelToken
```

或一条命令：

```powershell
python scripts/set_production_url.py `
  --public-url https://cursor-reports.你的域名.com `
  --tunnel-token "粘贴TunnelToken"
```

### 3. 安装 cloudflared（若未装）

```powershell
powershell -File scripts/install_cloudflared.ps1
```

### 4. 启动并注册开机自启

```powershell
powershell -File scripts/install_production_task.ps1
powershell -File scripts/start_production_stack.ps1
```

外网验证：

```powershell
curl https://cursor-reports.你的域名.com/api/v1/health
```

### 5. 重新打包发给同事

```powershell
python scripts/set_production_url.py --public-url https://cursor-reports.你的域名.com
powershell -File scripts/build_colleague_package.ps1
python scripts/export_member_credentials.py
```

## 二、没有 Cloudflare 域名时

请 **IT** 在公网服务器（如 `admin.vertu.cn`）加 Nginx 反代到内网 `http://10.100.0.176:8080`，示例：

```nginx
location /cursor-reports/ {
    proxy_pass http://127.0.0.1:8080/;
    proxy_set_header Host $host;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

则 `REPORT_API_URL=https://admin.vertu.cn/cursor-reports`

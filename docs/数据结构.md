# 日报数据结构

同事 Skill **整理后 POST** 的 JSON 结构。服务端 `api/schemas.py` 与本文档一致。

---

## 顶层字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `user` | string | ✅ | 英文名，须与 API Token 绑定用户一致 |
| `date` | string | ✅ | `YYYY-MM-DD` |
| `generated_at` | string | | ISO8601 时间 |
| `total_sessions` | int | | 今日会话数 |
| `total_turns` | int | | 今日对话轮次 |
| `daily_summary` | string | ✅ | **主管看的摘要**（Skill Step 2 精炼） |
| `key_topics` | string[] | | 关键词 |
| `all_files_modified` | string[] | | 涉及文件列表 |
| `sessions` | object[] | | 会话明细 |

**不要提交** `_raw_sessions`（内部字段，POST 前删除）。

---

## sessions[] 每条

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 会话 ID |
| `summary` | string | 会话标题/摘要 |
| `topics` | string[] | 主题标签 |
| `turns` | int | 轮次 |
| `tools_used` | string[] | 使用的工具 |
| `files_touched` | string[] | 涉及文件 |
| `outcome` | string | `success` / `partial` / `failed` / `unknown` |

---

## 完整示例

```json
{
  "user": "Ivan",
  "date": "2026-06-12",
  "generated_at": "2026-06-12T17:30:00+08:00",
  "total_sessions": 2,
  "total_turns": 8,
  "daily_summary": "今日 Cursor 工作摘要：\n- 【报价单模板】调整 Excel 导出逻辑，已完成\n- 【客户名单核对】Read/Grep 比对 customers.csv，进行中",
  "key_topics": ["报价单", "客户名单", "Excel"],
  "all_files_modified": [
    "templates/quote.xlsx",
    "data/customers.csv"
  ],
  "sessions": [
    {
      "id": "a1b2c3",
      "summary": "报价单模板字段调整",
      "topics": ["报价单"],
      "turns": 5,
      "tools_used": ["Read", "Write", "StrReplace"],
      "files_touched": ["templates/quote.xlsx"],
      "outcome": "success"
    },
    {
      "id": "d4e5f6",
      "summary": "客户名单核对",
      "topics": ["客户"],
      "turns": 3,
      "tools_used": ["Read", "Grep"],
      "files_touched": ["data/customers.csv"],
      "outcome": "partial"
    }
  ]
}
```

---

## 同事侧三步对应

| 步骤 | 脚本 | 产出 |
|------|------|------|
| 1 解析 | `parse_transcripts.py --output .tmp/draft.json` | 原始结构化数据 |
| 2 整理 | Cursor AI 改 `draft.json` | 精炼 `daily_summary` |
| 3 提交 | `submit_report.py --file .tmp/draft.json` | POST → 数据库 |

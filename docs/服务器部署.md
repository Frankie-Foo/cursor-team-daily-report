# 服务器部署（3 步）

> API 和 PostgreSQL **放同一台服务器**。运维把 `global-pdca.vertu.cn` 反代到本机 `8080`。

---

## 第 1 步：把代码放到服务器

在 **数据库那台机**（或运维指定机）：

```bash
git clone https://github.com/Frankie-Foo/cursor-team-daily-report.git
cd cursor-team-daily-report
```

> 仓库已 **公开**，运维可直接 `git clone`。以下敏感文件 **不在 Git 里**，需从你笔记本拷贝：

| 文件 | 说明 |
|------|------|
| `.env` | DB 密码、公网 URL |
| `config/api_tokens.json` | 全员 Token |
| `config/odoo_user_ids.json` | uid 映射 |

也可选：你本机打离线包 `powershell -File scripts/build_server_deploy_package.ps1`，zip 内已含上述文件，私发给运维即可（无需再单独拷）。

---

## 第 2 步：一键部署

**Windows 服务器：**

```powershell
cd D:\cursor-team-daily-report
powershell -File scripts/deploy_server.ps1 -PublicUrl https://global-pdca.vertu.cn
```

**Linux 服务器：**

```bash
chmod +x scripts/deploy_server.sh
./scripts/deploy_server.sh https://global-pdca.vertu.cn
```

**长期运行 API：**

```powershell
# 推荐：注册计划任务，开机自启
powershell -File scripts/install_api_server_task.ps1 -StartNow

# 或临时手动（关窗口即停）
powershell -File scripts/run_api_server.ps1
```

```bash
# Linux
python3 -m uvicorn api.server:app --host 0.0.0.0 --port 8080
```

---

## 第 3 步：让运维配域名

把 `config/nginx-global-pdca.example.conf` 发给运维，要求：

```
https://global-pdca.vertu.cn  →  http://127.0.0.1:8080
```

验证：

```powershell
curl https://global-pdca.vertu.cn/api/v1/health
# 期望: {"status":"ok","service":"cursor-team-daily-report"}
```

## 发给同事

部署成功后：

```powershell
powershell -File scripts/build_colleague_package.ps1
python scripts/export_member_credentials.py
```

- 群发：`package/colleague/cursor-team-daily-report.zip` + 使用说明
- 私发：每人 Token（见 `config/member_credentials.md`）

同事 `.env`：

```env
REPORT_API_URL=https://global-pdca.vertu.cn
REPORT_API_TOKEN=个人Token
```
